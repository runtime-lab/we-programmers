# 7장 니가스와 달: 첫 번째 OOPL

- 객체지향 프로그래밍(OOP)은 욜레-요한 달과 크리스텐 니가드라는 두 인물의 협업을 통해 탄생한 결과물이다.

### 크리스텐 니가드

- 1956년, 니가드는 “Theoretical Aspects of Monte Carlo Methods”(몬테카를로 기법의 이론적 탐구) 논문을 발표한다.

### 욜레-요한 달

- 올레는 1952년 병역 의무의 일환으로 NDRE에서 파트타임으로 일했고, 졸업 후에는 정규직이 되었다.
  - 당시 올레의 NDRE 상관이 바로 크리스텐 니가드이다.

### SIMULA와 객체지향(OO)

- SIMULA 67은 최초의 객체지향 프로그래밍 언어이다.
  - C++를 만든 비야네 스트롭스트룹과 스몰 토크(Smalltalk)를 만든 앨런 케이에게 영향을 주었다.
- NDRE에서 페란티 머큐리 컴퓨터의 프로그래머로 일했다.
- 달은 NDRE에서 머큐리 컴퓨터용 어셈블러를 작성했다.
  - ALGOL의 초기 보고서에 영감을 받아 머큐리 자동 코딩이라는 고급 언어용 컴파일러를 개발하기도 했다.
- 니가드는 머큐리 컴퓨터를 위한 여러 가지 시뮬레이션의 개념을 수학적 언어로 정형화하여 좀 더 이해하기 쉬운 형태로 만들려고 고민 했다.
  - 니가드는 자신의 고민을 적은 노트를 정리하여 몬테카를로 컴파일러(Monte Carlo Compiler)라는 아이디어를 정립한다.
  - 몬테카를로 컴파일러에 대한 니가드의 아이디어는 두 가지 유형의 데이터 구조에 기반을 두고 있다.(이산 이벤트 네트워크)
    - 커스터머(customers)라 불리는 구조로 여러 속성 데이터를 수동적으로 담고 있는 저장소이다.
    - 스테이션(stations)으로 내부에 큐를 가지고 있어, 큐에 대기하고 있는 커스터머를 처리한 후 다른 스테이션의 큐로 넘겨 주는 역할을 한다.
  - 커스터머가 스테이션의 큐에 삽입되는 모든 과정은 이벤트로 트리거되는 방식이 사용된다.

#### SIMULA I

- SIMULA 컴파일러 개발에서 스택(stack) 문제 때문에 순조롭게 진행되지 못한다.
  - 당시 컴퓨터에는 스택이라는 개념 자체가 없었고, 스택 포인터로 사용할 수 있는 인덱스 레지스터도 없었다.
- 그들은 블록({})을 스택으로 관리하지 않고 힙과 가비지 컬렉터를 이용해서 더 유연하게 만들었다.
  - 힙의 객체를 올리면서 객체의 생존 기간을 더 유연하게 조정할 수 있다.
  - 이로 인해 해동을 내포한 능동적인 객체가 가능해졌다.

```java
// 수동적인 데이터 구조의 예
class Customer {
	String name;
	int age;
}

// 능동적인 객체 구조의 예
class Customer {
	String name;
	int age;
	void checkIn() {
		System.out.printIln(name + " has checked in.")'
	}
}
```

- 스테이션과 커스터머는 결국 시뮬레이션 안에서 동시에 동작할 수 있는 다양한 종류의 구성 요소 중 하나일 뿐이었다.
  - SIMULA가 단순히 몬테카를로 시뮬레이션 전용 언어를 넘어 범용 프로그래밍 언어로 발전할 수 있다는 가능성을 ㅂ깨달았다.
  - 커스터머와 스테이션이라는 용어 대신 일반화된 액티비티(activities)가 사용되었다.
- 1967년 5월, 달과 니가드는 클래스와 서브클래스 선언에 관한 논문 발표한다.
  - 인스턴스와 객체라는 용어, 가상 프로시저 개념도 이 논문을 통해 도입되었다.
- 다른 컨퍼런스에서 타입과 클래스의 동일성에 대한 내용도 발표한다.
