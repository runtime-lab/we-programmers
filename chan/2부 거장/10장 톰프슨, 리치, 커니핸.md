# 10장 톰프슨, 리치, 커니핸

- 1968년부터 1976년 사이 유닉스(Unix)와 C의 등장은 컴퓨팅 산업의 역사에서 가장 중요한 사건이다.
  - C언어와 유닉스는 서로 영향을 주며 발전했다.

#### 타임셰어링 시스템

- 멀틱스는 메모리 매핑, 동적 링크, 파일을 메모리에 매핑하거나 메모리를 파일에 매핑하는 기능 등이 포함되어 있었다.
  - 모든 타임셰어링 시스템을 대체할 시스템을 만들려고 설계해서 지나치게 범위가 커지게 했다.
  - 벨 연구소는 결국 멀틱스에서 손을 떼기로 한다.

### 유닉스

- 최초의 유닉스는 바이트를 기반으로 한 운영체제가 아니었다.
  - PDP-7의 18비트 워드 길이가 모든 것을 제한했기 때문이다.
  - 파일은 18비트 워드의 연속적인 형태로 만들어졌다.

### PDP-11

- 제한적인 PDP-7 메모리 등의 제약으로 PDP-11 컴퓨터로 바꾸었다.
- PDP-11은 메모리가 바이트 단위를 기반으로 하고 있으며, 내부 아키텍처는 16비트 구조였다.
  - 바이트 두 개가 모여 하나의 워드를 이루고, 각 워드는 하위 바이트를 먼저 저장하는 리틀 엔디안 방식으로 저장되었다.
- 명령어는 레지스터를 1 또는 2만큼 미리 감소(pre-decrement)시키거나 실행 후 증가(post-increment)시키도록 만들 수 있었다.
  - 추후 C언어의 `i++`와 `--i` 표현식을 구현하는데 유용하게 쓰인다.
- 톰프슨은 매킬로이의 논문에서 ‘프로그램들이 마치 정원의 호스처럼 연결될 수 있어야 한다.’라는 내용에서 아이디어를 얻어, 유닉스 파이프(Unix pipes)를 구현한다.
  - 기존 유닉스 애플리케이션들은 전부 파이프로 연결되어 일종의 필터처럼 동작할 수 있게 되었다.

### C언어

- C언어는 TMG라는 언어에서 시작된다.
  - TMG는 TransMoGrifiler(변형기)의 약자로 맥클루어가 만든 언어이다.
  - 맥클루어가 연구소를 떠나면서 TMG 소스 코드를 가져가서 맥킬로이는 TMG를 다시 작성해서 PDP-7 어셈블러 코드를 생성했다.
- 당시 톰프슨은 컴퓨터라면 반드시 포트란 언어를 쓸 수 있어야 한다고 생각해서, TMG를 이용해 포트란을 작성했다.
  - 하지만 PDP-7 유닉스의 사용자 할당 용량인 4K를 초과해서 일부 기능을 하나씩 제거하고 컴파일 하면서 4K 용량에 맞는 컴파일러르 만든다.
  - 결과적으로 포트란과의 차이가 생기고 멀틱스 언어인 BCPL과 더 유사해서 B언어라고 이름을 붙힌다.
- 톰프슨은 B에 더 많은 기능을 추가하고 싶었지만 용량 제한을 초과해서 인터프리터 언어인 특징을 이용해 파일에 저장해 둔 채 가상적인 방식으로 실행했다.
  - 가상 실행 방식은 비록 느리지만 컴파일러를 다시 4K로 줄이는데 유용했다.

> **작성자 해석**: 코드를 메모리에 다 올리지 않고 필요할 때마다 읽어서 실행함.
> 컴파일 언어라면 전체를 기계어로 만들어 메모리에 올려야 하지만, 인터프리터라면 파일에 한 줄씩 읽고 실행이 가능하다.

- 톰프슨은 ‘세미콜론’을 활용 for 루프와 `++`와 `+=` 스타일의 연산자를 추가한다.
- PDP-11이 도입되고 유닉스가 가동되자 리치는 B 언어를 PDP-11에 이식하기로 결정한다. 하지만 PDP-11의 아키텍처 문제에 부딪힌다.
  - 당시 B 언어는 명시적인 타입이 없었고, 모든 것이 암묵적 타입의 워드(word)였다.
  - PDP-7에서는 그 워드가 18비트 정수였어서 바이트 기반의 PDP-11에서 제대로 된 연산을 수행하거나 포인터를 담기에 너무 작았다.
    - 리치는 B언어에 타입을 추가하기로 했다. (char, int)
    - 배열과 포인터 처리를 통합한다.
  - 추후 리치는 C언어라고 이름을 붙힌다.
- 리치는 `struct`을 추가하면서 유닉스를 C로 이식하는 문제를 해결한다.
  - 여러 타입의 데이터를 논리적으로 하나의 단위로 묶어서 다룰 수 있게 해주는 도구였다.
