### C언어
	1. B 언어 -> P코드 -> C 언어로 진화했다.
	2. B 언어는 PDP-7에서 인기가 있었지만 성능적인 압박이 꽤나 심했다고 한다.
	3. 유닉스 기반의 PDP-11에서 B 언어를 이식하려 시도헀지만 아키텍처 문제로
		실패했다.
		3.1 B 언어에는 명시적인 타입이 없었는데 PDP-11는 바이트 기반 컴퓨터로
			바이트는 포인터를 담기에 작았다.
		3.2 그에 대한 해결책으로 char와 int 타입이 만들어진다.
	4. 이후 포인터 처리를 통합하고 타입 체계를 확장한 문법이 C언어다.
	5. 유닉스를 C로 이식하려 했지만 세 번 실패했다고 한다.
		5.1 실패의 원인이 언어에 있다고 주장하며 다양한 리펙토링이 진행된다.
		5.2 구조체를 추가한 시점에서 유닉스를 성공적으로 이식 할 수 있었다.


### 느낀 점
	1. C 언어 이전 세대에 B와 P가 있는 것은 몰랐던 일 이다.
	2. 사실 C 언어 자체도 깊게 본 건 아니지만 입문을 C로 시작하기는 했다.
		2.1 당시에는 일단 언어 하면 C 언어가 근본이고 C를 잘 하면 나머지는 다 날먹할 수 있다는 의견이 주류였다..
	3. 중간 단계에서 언어가 뽑아낼 수 있는 성능을 하드웨어가 따라오지 못 하는 구간들이 보이기
		시작하는 것으로 보면 양쪽은 무조건 상호 보완의 관계로 보인다.
	4. 하드웨어에 대해 크게 고민해본 적은 없다만 한 번 해보는 것도 나쁘지 않겠다.
