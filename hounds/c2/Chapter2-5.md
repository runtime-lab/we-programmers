# Chapter 2-5 거장 / 에츠허르 다익스트라

### 첫 번째 컴퓨터 과학자
    1. 세마포어의 개념을 처음 고안했다.
        1.1 일종의 동시성 제어를 위한 동기화 원시 타입이다.
        1.2 현대의 세마포어는 다익스트라가 제안한 고전적인 동기화 기법을 발전시킨 형태로
            단순한 Lock 보다는 동시성 환경에서 자원에 대한 접근 제어 기법이라고 보는 것이 옳다.
    2. 추상화의 중요성에 대해 강하게 주장했다고 한다.
        2.1 실제 현대의 프레임워크 또는 라이브러리는 대부분 파사드를 사용하고 인터페이스를 기본으로 가져간다.
            이로 미루어보아 대단한 선견지명이다.

### 생애
    1. 네덜란드 출생 교육자 집안의 아들로 태어났다.
        1.1 부모의 주 종목이 수학과 과학이었다.
    2. 놀랍게도 법학에 꿈이 있었지만 결국 이론 물리학을 전공했다.
    3. 1951년부터 프로그래밍에 대한 학습을 시작했다고 한다.
        3.1 의외로 영어가 서툴렀다는 서술이 있다.
    4. 현업으로서 첫 업무는 ARRA의 명령어 집합을 설계하는 일 이었다.
        4.1 이로 인해 프로그래밍 언어의 설계에 대한 관심이 생겨서 프로그래밍 언어를 연구하기 시작했다.
    5. "창의적이면서 정확해야 한다."는 특징을 좋아했다고 한다.
        5.1 물론 중간 과정에 자신이 프로그래밍과 맞지 않는다는 의심을 하기도 했다.

### ARRA
    1. 프로그램 내장 방식의 전자 기계식 컴퓨터다.
        1.1 이 당시에도 부품에 진공관이 들어있었다.
    2. 개발 과정에서 제대로 동작하지 않았다고 한다.
    3. 사실 ARRA 1이 대차게 망해서 새로 개발 했다.
        3.1 이 사실을 알리고 싶지 않아 ARRA 2를 그냥 ARRA라 불렀다고 한다.
        3.2 주요 변경점은 전기 기계식 계전기가 아니라 전자식으로 새로 설계되었다.
    4. 1953년에 완성되었다.
        4.1 심지어 전작과는 다르게 제대로 작동했다고 한다.
    5. 당시에는 기계의 작동음이 디버깅 수단 중 하나였다고 한다.
        5.1 레지스터 중 하나에 스피커를 연결하고 그 소리를 듣는 방식이었다고 하는데
            기계가 정상 동작 할 때와 무한 루프에 빠졌을 때 소리가 다르다고 한다.
            아마 계속해서 같은 소리가 반복되면 무한 루프로 판단하는 것 같다.
    6. 이례적으로 10진수가 아닌 2진수를 사용했다고 한다.
    7. 다익스트라가 작성한 ARRA 2의 명령어는 비교적 단순했다.
        7.1 8진수나 16진수 개념이 이 당시에는 상용화 되어있지 않아 명령 작성에 10진수를 사용했다고 한다.
        7.1 ex) 0/n replace (A) with (A) + (n)
    8. 드럼 메모리가 64개의 채널로 구성되어 있었다. 각 채널에는 16개의 워드가 저장 되었다.
    9. 간접 주소 지정이 없었다고 한다.
        9.1 아마 선언적으로 변수 재참조를 할 수 없었다는 의미로 보인다.
        9.2 호출과 반환 주소를 기억하는 기능도 없었다고 하는데 마지막 명령에 이동 할 분기를 직접 지정해야 했다고 한다.
            지금으로 치면 return 구문이 없다는 의미로 보인다.
        9.3 여기에 영향을 받은 다익스트라는 호출 가능한 구조의 효율에 대하 강조했다.
            같은 명령어들을 프로그램 곳곳에 복사하는 보일러 플레이트를 없애고 한 번만 정의해서 실행하는 것에 대한 효율을
            언급했는데 지금으로 따지면 싱글톤 패턴 + 메소드 호출을 의미한다.

### ARMAC
    1. 1956년에 등장한 ARRA 계열의 컴퓨터다.
        1.1 초당 명령어 1,000 개를 실행 할 수 있었다.
    2. 하드웨어의 발전이 힘을 실어줘 다익스트라 알고리즘이 실현된다.
        2.1 흔히 알고 있는 최단 경로 알고리즘인 통칭 다익스트라 알고리즘으로 통한다
        2.2 커피 마시다가 20분 만에 설계한 알고리즘이라고 한다.
            2.2.1 우와
    3. 당시에 부품으로 사용 될 구리가 아주 귀한 재료였다고 한다.
        3.1 구리 소모량을 줄이기 위해 최소 신장 트리를 창안했다고 기술하는데 사실 다익스트라가 MST를 창안하지는 않았다.
            3.3.1 최소 신장 트리의 출발은 사실상 보루프카의 1926년 해법이다.
            3.3.2 다만, 다익스트라는 연구된 MST를 현대에 맞게 재발견하고 보급되는 것에 영향을 줬다.
            3.3.3 최소 신장 트리의 예를 들면 n개의 정점을 모두 연결 할 때 가장 이동 수가 적은 간선을 택하는 방식이다.

### ALGOL / X1
    1. 당시에는 하드웨어와 결합되는 언어가 프로그램 실행의 효율성 확보에 필수적이라는 인식이 있었다고 한다.
        1.1 범용 언어가 실행 속도는 느리지만 시간 단축, 실수를 방지하는 이점이 있다는 이견 또한 함께 존재했다.
    2. ALGOL의 목표는 수학적 표기법을 사용해 알고리즘을 문서로 출판하고 나아가 기계어 프로그램으로 변환 할 수 있도록
       하는 것이 었다고 한다.
        2.2 초기 표기법을 배커스 정규형이라 명명했다.
        2.3 이후 1960년 버전의 ALGOL이 정의 된다.
    3. 동시간 대 다익스트라 측은 X1 컴퓨터 제작을 진행 중 이었다.
        3.1 부트 프로그램과 초기 어셈블러가 탑재된 컴퓨터인데 최초로 하드웨어 인터럽트가 탑재되었다.
        3.2 인덱스 레지스터가 실려있어 포인터 사용이 가능했다고 한다.
        3.3 마이크로초 단위로 진입해 초당 1만 개 이상의 명령 실행이 가능했다.
    4. 다익스트라 측이 ALGOL 언어 정의 작업에 합류 후 명세가 완성되자 광속으로 X1의 컴파일러를 만들어낸다.
        4.1 놀랍게도 컴파일러를 작성해 본 경험이 없었고 사용 방향이 정립되지도 않은 기계였다고 한다.
            4.1.1 우와
    5. 이중 프로그래밍이라는 기법을 사용했는데 두 사람이 각자 만든 후 비교하는 방식이었다고 한다.
        5.1 지금의 페어 프로그래밍 개념으로 보인다.
    6. ALGOL 구현 팀이 기능의 포함, 제외를 두고 싸우고 있을 때 다익스트라 쪽에서 6주만에 전부 다 구현했다고 한다.
        6.1 6주 만에 선구자들을 범부들로 만들었다.
        6.2 포기 했던 재귀도 구현했다고 한다. 구현해서 슬쩍 집어넣었다고 하는데 심지어 투표로 제외가 결정 된 기능이었다고...
        6.3 구현 팀 입장에서는 굉장히 띠꺼웠을 것으로 생각된다.
        6.4 성공의 비결은 언어와 기계 사이에 추상화 경계를 둔 것이라고 한다.
            6.4.1 오늘 날의 개념으로 VM을 사용했다고 한다. 대신 좀 느렸음.
    7. 재귀의 중요성을 계속해서 강조했는데 자후 PDP-11의 등장이 이를 증명했다.
        7.1 PDP-11은 16비트 미니 컴퓨터 시리즈다.

### 컴퓨터 과학
    1. ALGOL 컴파일러가 완성되고 다익스트라는 어느 정도 하드웨어의 구속에서 해방 될 수 있었다고 한다.
    2. 다익스트라는 THE 멀티프로그래밍 시스템을 연구했는데 X8 모델에서 실행되었다.
        2.1 X8은 당시 멀티 프로세싱 실험을 위한 최적의 플랫폼이었다.
    3. THE 시스템의 아키텍처는 매우 현대적인 구조를 가졌다.
        3.1 레이어 0 : 프로세서 할당 기능 (현재의 OS 스케쥴링)
        3.2 레이어 1 : 메모리 제어
        3.3 레이어 2 : 시스템 콘솔 제어 (I/O 프로세스 보장)
        3.4 레이어 3 : I/O 동작 + 버퍼링 제어
        3.5 레이어 4 : 프로그램 실행 계층

### 세마포어
    1. 프로세스 경쟁 상태를 제어하기 위한 추상화 기법
    2. P(Acquire) / V(Release) 연산으로 구성
    3. 임계 구역
        3.1 공유 자원을 조작 할 수 있는 프로그램의 특정 부분
        3.2 단독 실행이 보장되는 구역으로 동시성 제어 영역
    4. 불가분 연산
        4.1 인터럽트 발생 전 반드시 전부 실행되어야 하는 연산
        4.2 세마포어의 증감 연산은 반드시 실행 되어야 함.
