# Chapter 2-4 거장 / 존 배커스

### 1. IBM 701
    1. 완전한 전자식 컴퓨터다.
    2. 프로그램 내장 방식과 1024 * 72 비트 메모리를 지원한다.
    3. 초당 약 14,000 개의 명령어를 실행 할 수 있다.
    4. 동일하게 천공 카드를 사용했다.

### 2. 스피드코딩
    1. 부동소수점 연산을 지원하는 인터프리터
    2. 각 명령은 수학 연산과 논리 연산 둘로 구성된다.
    3. 인덱스 레지스터를 통한 간접 주소 지정으로 배열 순차 처리에서 프로그래머의 명령어 수정이 사라졌다.
    4. 3가지 레벨의 간단한 로깅/트레이싱을 지원한다.
    5. 여기까지도 천공 카드에 의존했다.
    6. 가장 큰 문제는 인터프리터의 속도가 매우 느렸다는 점 그럼에도 프로그램 작성 시간을 몇 시간으로 단축 시킨 것은
       아주 큰 발전이었다.

### 4. IBM704 / 포트란
    1. 스피드코딩을 통해 추상화가 프로그래머의 생산성에 큰 영향을 미친다는 것을 인지
    2. IBM 704 컴파일러 개발을 시작하면서 원칙을 세웠다.
        - 효율적인 코드를 생성하는 컴파일러를 만들자.
        - 가장 중요한 것은 언어를 통해 얼마나 빠르고 효율적으로 실행되는 프로그램을 만들 수 있는가
    3. 그 문법 자체는 IBM 704에서만 효율적으로 동작한 언어
    4. 변수명을 한 글자 또는 두 글자로 만들 수 있었는데 자료형 지정이 변수명의 첫 글자를 따라갔다고 한다.
        - N으로 시작하면 정수형, 그 외에는 실수형
    5. 행렬 연산이나 분기 처리 등 내구 구조적으로는 강했지만 입/출력 기능에 큰 고려를 하지 않았다.
    6. IBM 704 역시 주로 천공 카드에 의존했고 이 시대에는 종이 테이프나 천공 카드가 곧 프로그램이었다.

### 5. 실행 프로그램
    1. 초기에는 컴파일러가 IBM 704 실행 프로그램을 생성할 때 소스 코드를 한 번 만 읽는 방식을 원했다고 한다.
        - 아마 지금으로 표현하면 클래스 로더가 전체 클래스를 퍼다 올리는 것과 비슷한 개념으로 보인다.
    2. 이를 위해 팀을 각 파트로 나눠 구현을 진행했다.
        - 파서 -> 컴파일 대상 분리, 분석 처리 + 테이블 형 변환
        - 코드 생성 -> 파서가 생성한 테이블로 기계어 코드 생성
        - 중간 통합 파트 -> 코드 생성 출력 통합, 구조화 레이어에 일관된 입력 제공
        - 프로그램 구조화 -> 코드를 하나의 진입점, 종료점을 가지는 블록으로 분할 + 블록에 대한 몬테카를로 분석
            - 몬테카를로 분석 : 무작위 추출된 난수를 이용하여 원하는 함수의 값을 계산하기 위한 시뮬레이션 방법
        - 태그 분석 -> 인덱스 레지스터 수 최적화(IBM 704의 스펙에 맞춘 최적화) + 재배치 가능한 어셈블리 코드로 변환 
        - 백엔드 -> 최종 이진 코드 생성 + 천공 카드 형태 출력
    3. 프로그래밍 역사에 한 획을 그은 포트란의 탄생이다.

### 6. 배커스 정규형
    1. 미국 컴퓨터 학회가 범용 프로그래밍 언어의 가능성을 탐색하기 시작
    2. 위원회가 만드려고 했던 언어는 국제 대수 언어로 초기 명세서가 모두 영문으로 작성된 문서
        - 문서 일관성에 대한 비판이 있었다.
    3. 범용 프로그래밍 언어를 기술할 수 있도록 배커스가 고안한 생성 규칙
        - 생성 규칙들을 연결해 언어의 문법 구조를 설명할 수 있는 메타 언어다.

### 7. 느낀점
    1. 이 시대가 현대적인 프로그래밍으로 넘어오는 과도기였다는 생각이 든다.
    2. 년도를 보면 6.25 전쟁과 대체로 맞물리는 시기인데 그 시기에 다른 대륙의 역사 속에서는 이런 발전들이 있었다는 점을
       알아 봤다는 것이 흥미롭다.
    3. IBM 704의 컴파일러에 내부 책임을 나누어 구현했다는 점은 인상 깊다.
       지금의 컴파일러와 대조해 포트란 컴파일러가 완성형 구조를 제시했다고 볼 수는 없다.
       다만, 지금의 컴파일러가 각 레이어를 나눠 분업을 수행하는 것과
       고수준 언어를 분석하며 기계 친화적인 코드로 내리는 과정이 아직 명맥을 유지하고 있기 때문에 
       한 시대에 머무른 것이 아닌 후대의 컴파일러가 채택하는 기본 분업 구조의 원형을 제시했다는 것에서 전설적인 유산이라 표현하고 싶다.
