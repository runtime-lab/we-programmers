### 1. 계산표
- 계산표란?
  - 어떤 함수값, 계산 결과를 미리 수치로 정리해 서면 제작한 표
  - 과거 종이책, 서면표, 천문표, 로그표, 삼각함수표 등 다양한 형태로 널리 제작됨.
  - 핵심은 형태보다 "미리 계산된 수치표" 라는 점 
  
  <br>

- 계산표의 제작
  - 기본
    - 복잡하고 추상적인 개념을 계산 가능한 형태로 변환
    - 다항식으로 정확히 계산 할 수 없는 원소에 대해 근사치로 접근
    - 예시
      - y축을 기준으로 1과 -1 사이에서 파동하는 x축의 주기가 2𝝅인 사인 곡선
      - 이 사인 곡선 위에 y= -0.16666x^3 + x 라는 그래프를 겹쳤을 때 x가 0에 가까울 때는 꽤 근사한 편
      - 이러한 방식으로 y= -0.0001984x^7 + 0.00833^5 - 0.1666x^3 + x의 경우 더 근사한 형태를 가짐.
        - -𝝅/2 ~ 𝝅/2 구간에서 아주 정밀 (소수 넷 째 자리까지 근사)
      - 이 계수들은 테일러 전개에서 나옴. <br> <br>
  - 유한 차분법 (한 번에 이해가 안 됨..)
    - 일정 간격으로 떨어진 값들 사이의 차를 이용해 변화 규칙을 분석하는 방법.
    - 예를 들어, 0 ~ 𝝅/2까지 각 1초마다 사인 값을 계산한다고 가정<br> 이 구간에는 1초가 총 32만 4,000개 존재
    - 34만 번의 작업을 수행하지 않기 위해 유한 차분법을 사용
    - 기본 원리
      - | x | x^2+3x-2 | d1 | d2 |
        |---|----------|----|----|
        | 1 | 2        |    |    |
        | 2 | 8        | 6  |    |
        | 3 | 16       | 8  | 2  |
        | 4 | 26       | 10 | 2  |
        | 5 | 38       | 12 | 2  |
      - f(x) = x^2+3x-2 (2차 다항식)
      - d1 = 1차 차분
      - d2 = 2차 차분
      - 차분이란?
        - "인접한 결과값"들의 차이
        - 왜 d1은 x = 1인 케이스에 존재할 수 없나
          - 차분은 "인접한 결과값"들의 차이로 결과값이 하나만 존재하면 있을 수 없음.
          - d1 = 현재 행 값 f(x) - 이전 행 값 f(x-1) -> 이전 행이 없으니 계산 불가
        - 왜 d2는 x = 3부터 생성되나
          - d2는 d1의 차분으로 x = 2에서 최초의 d1 생성, x = 3에서 첫 번째 차분 수립
        - n차 다항식의 차분은 n차 차분부터 선형적 구조 형성
          - 예시로 표의 2차 차분의 값은 2로 상수화
          - 그 이후의 차분은 모두 0
      - 차분 응용 예시
        1. f(6) -> (마지막 1차 차분(12) + 마지막 2차 차분(2) = 14) + 마지막 함수 값(38) -> 52
        2. f(7) -> (마지막 1차 차분(14) + 마지막 2차 차분(2) = 16) + 마지막 함수 값(52) -> 68
        3. 다음 함수 값 추출 -> 값으로 1차 차분 추출 -> 값으로 2차 차분 추출 순서로 드르륵 가능
  - 이 방식이 자동 컴퓨터가 등장하기 전 수학적 계산표를 만들던 방식 <br> 원청 수학자(다항식 도출) -> 하청 수학자(각 함수 구간에 대한 차분표 작성) -> 하하청 수학자(차분표를 사용한 실제 계산표 작성)
    - 그래서 이 사람들을 불렀던 명칭이 컴퓨터라는 기록이...


### 2. 차분기관의 탄생
1. 배경
   -  배비지와 그의 친우인 존 허셜은 천문학회에서 사용할 계산표를 검토하는 업무를 수행.
   -  계산표는 각각 12자리 이상의 숫자가 수 천 개 이상 포진
   -  수행 방식은 서로 숫자를 낭독하고 확인 후 불일치 해결 및 주석 처리
   -  작업 방식에 대해 이를 "증기를 사용해 처리할 수 있으면 좋겠다"는 배비지의 아이디어(수요) 발생
     - 위에서 설명한 "컴퓨터" 들을 신뢰할 수 있는 기계로 대체
   -  또한 계산 기계를 사용하는 것이 과학 발전에서는 필수일 것이라는 비전을 봄.

   <br>
2. 차분기관
   - 아주 거대한 덧셈 기계
   - 최대 20자리 십진수까지 저장 가능한 레지스터를 6개 장착
     - 6차 차분까지 표현 가능 -> 6차 다항식까지 컨트롤 가능
   - 약 25,000 개의 부품으로 구성 (정밀함 GOAT를 꿈 꾸며 만들어 냄.)
   - 배비지의 메커니즘
     - 차분기관에서 올림 처리를 할 때 구조적으로 톱니바퀴에 강한 힘이 필요하고, <br> 기계를 조작하는 사람 팔이 기계보다 먼저 고장날 것 같음.
     - 각 톱니바퀴에 올림 정보(있음, 없음)를 지정하는 레버를 만들고 <br> 기계 작동 주기 안에서 특정 시점에 순차적으로 전달하는 방식으로 기계와 사람 모두의 부하를 줄임.
     - 물리적인 형태지만 플로우를 가진 일련의 과정을 만들어 냄. <br> <br>
3. 기계식 표현법
   -  차분기관이 탄생한 후 배비지는 기계의 동작을 표현하기 위한 표기법 체계를 창안
   -  배비지는 이 표기법을 "상호 작용하는 부품들의 보편적인 추상 언어"로 생각
   -  어쩌면 이것도 하나의 프로그래밍 언어로 볼 수 있지 않을까..?  <br> <br>

### 3. 차분기관의 실패
1. 사업
   -  배비지는 천문학회, 런던 왕립학회 정회원, 의회에서 인맥이 좀 있던 편
   -  정부 지원 하에 차분기관을 제작할 수 있는 위탁 사업 시작
   -  17,500 파운트 (현재 기준 한화 약 51억) 대규모 투입 사업을 10년 간 진행
     - 계약자와 야차, 작업 중단, 예산 초과<br><br>

2. 중단
   -  10년 동안 계속 돈만 까먹는 사태가 발생하자 조지 비델 에어리가 정부에 상소문 올림. (님들 돈 낭비 중임) <br> -> 영국 정부도 더 이상은 안될 것 같다고 판단 후 중단 선언
   - "비용이 무한정 늘어날 것으로 보인다.", "성공 여부가 불확실하다.", "지출 규모가 지나치고 산정이 안 된다."
   -  사실 완성되지 못한 이유가 당대의 금속 가공 기술이 정밀하지 못 했기 때문이라는 말이 있지만 증거가 없다고...
