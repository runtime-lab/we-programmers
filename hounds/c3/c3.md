# Chapter 3 과거

### 1960 ~
	1. 반문화의 시대
	2. ECP-18이 교육을 위해 미국 전역에 보급되던 시기였다.
	3. 저자의 아버지가 프로그래밍 교육에 대한 지원이 많았다는 서술이 있다.
    4. 주로 작가의 성장기에 프로그래밍을 시작하게 된 계기를 서술 한다.

### 1970 ~
	1. 주로 작가의 업무 환경에 대해 이야기한다. A.S.C. 에서 일 했다고 한다.
		1.1 당시에 사용된 컴퓨터는 DATANET-30 이다.
	2. IBM의 장비를 다루는 업무를 수행했다.
	3. 소스 코드 관리 방식에 대해 서술했는데 압정으로 마일스톤을 표현했다고 한다.
	4. 조금 더 진보된 코드들이 탄생하기 시작한다.
		4.1 주로 자기 테이프 기반이 많이 등장한다.

### 1980 ~
    1. 네트워크의 성장 시기로 www가 탄생했다.
    2. ratfor이라는 C와 유사한 언어가 존재했다고 한다.
        2.1 유닉스 플랫폼에서 사용하는 표준 도구들을 구현할 수 있었다고 하는데
            ls, cp, grep 같은 리눅스에서도 사용하는 명령어들이 있는 것으로 보아
            cli 개발 언어였던 것으로 보인다.
    3. 인프라 아키텍처가 성장하는 시기였다.
        3.1 지금의 공인망이 동작하는 방식이 슬슬 드러나기 시작한다.
    4. 지금의 ARS와 비슷한 개념이 등장한다.
        4.1 E.R이라고 서술되어있다.
        4.2 이후 VRS(Voice Response System)도 등장한다.
    5. 매킨토시용 Aztec-C 컴파일러를 구입했다고 서술해뒀는데 이 시기에도 컴파일러는
        물리 매체에 담겨서 배포되던 시대였다.
    6. 회선의 정전 용량을 통해 최선의 단선 지점을 추정한다는 서술이 등장한다.
        6.1 전방에 있던 시절 이와 비슷한 메커니즘을 광망에서 실제로 본 적이 있다.
            조금 다른 점은 광망은 빛을 통해 단선을 진단한다는 것 이다.
            철책에 걸려있는 그물을 광망이라 불렀는데 SK와 삼성에서 광망 사업을 주관했다.
            아마 우리는 SK를 옆동네는 삼성을 썼던 걸로 기억한다.
            파이버 헌터라는 데스크탑 앱이 같이 제공됐었는데 데스크탑으로 돌리지는 않고 소초마다 서버실이 따로 있었다. 
            상시 에어컨 풀가동이라 여름에는 자물쇠 따고 들어가서 앉아있었던 기억이 난다.
            중/근거리 카메라, TOD 화면 관제와 광망 절곡 경보, 절단 경보를 같이 수신할 수 있었다.
    7. 유한 상태 기계를 통해 전이에 대해 표현한다.
        7.1 절차의 상태 전이를 텍스트로 기록했다고 서술하는데 일종의 원장 개념으로 보인다.

### 1990 ~
    1. 유닉스와 C언어가 지배적인 시대였다.
    2. 선 마이크로시스템즈에서 C++ 컴파일러를 만들었다.
        2.1 선 마이크로시스템즈는 JAVA를 만든 회사다. 이하 설명 생략 가능.
        2.2 기존의 C 언어로 작성된 코드가 호환 됐다고 한다.
    3. 다이어그램에 대한 이야기가 언급되는데 플로우 차트처럼 보인다.
        3.1 출발부터 어떤 단계를 거쳐 상태 관리,정책에 도달 하는지 표현한다.
    4. 객체지향 데이터베이스를 사용해 개발했다고 언급하는데 정확히 어떤 것인지 서술 되어있지 않다.
        4.1 초기 상용 제품으로는 Gemstone, VBase, GBase 등이 있다고 한다.
    5. 디자인 패턴이 등장하는데 자주 들어본 GoF가 서술 되어있다.
        5.1 디자인 패턴을 집필한 4인방이다.
    6. 객체 지향 10계명이 언급 되는데 SOLID 원칙의 전신이다.
        6.1 OCP 개방/폐쇄
        6.2 LSP 리스코프 치환
        6.3 DIP 의존성 역전
        6.4 ISP 인터페이스 분리
        6.5 SRP 단일 책임
        6.6 책에 서술된 10계명과 일부 항목에 차이가 있다.
    7. 닷컴 버블 시절에 대한 이야기가 언급 되어있다.
    8. 익스트림 프로그래밍
        8.1 개발 방법론으로, 빠르게 고객과 소통하며 개발할 수 있는 방법이다.
        8.2 빠른 릴리즈 텀으로 고객의 피드백을 통해 진화하는 것을 목표로 한다.
        8.3 추후 에자일의 요소로 채택된다.

### 밀레니엄
    1. 닷컴 버블이 붕괴하는 시대다.
    2. 동시에 에자일 얼라이언스가 결성 되었다.
    3. 저자가 클린 코드를 집필한 시기다.
        3.1 Java 하는 사람들한테는 성서로 취급 받는다.
    4. 에자일 운동이 본래의 목적을 벗어나기 시작한다.
        4.1 PM들이 프로젝트를 주도하기 시작한 것을 원인으로 언급한다.
    5. 새로운 언어가 탄생할 수록 생산성 향상이 0에 수렴한다고 언급한다.
        5.1 언어 뿐만 아니라 기술 전반적으로 한계치에 가까워지고 있다고 표현하는데
            실제로 과거 시대 수준의 혁신이 나타나지 않고 있는지 찾아 볼 필요가 있어보인다.

### 느낀 점
    1. 시대를 지나오며 겪은 저자의 일대기가 주 내용이다.
    2. 내용 안에는 실제 역사 속에서 이루어졌던 기술의 발전들이 회고처럼 적혀있는데
        꽤 익숙한 단어들도 자주 보인다.
    3. 세대가 발전했지만 프로그램의 본질은 계속해서 같은 명맥을 유지해왔다.
        어쩌면 내가 하는 생각과 행위들도 그 안에 함께 있는 지 점검해 볼 필요가 있어보인다.
